## Ralphthon 캡스톤 프로젝트

**Ralphthon**은 16주 강의의 최종 캡스톤 프로젝트다. Ralph 루프 방법론을 기반으로 **실제 소프트웨어 문제를 해결하는 자율 에이전틱 시스템**을 **개인으로 설계하고 구현**한다.

> **2026년 운영 방식**: 소규모 운영 방식을 반영하여 팀 프로젝트 대신 **개인 프로젝트**로 진행한다.

## 일정

| 주차 | 내용 | 마감 |
|------|------|------|
| 8주차 | 프로젝트 계획서 PR 제출 + 계획 발표 (중간고사 대체) | 2026-04-20 / 발표 04-21 |
| 13주차 | 착수 + 아키텍처 설계 문서 제출 | 2026-06-03 |
| 14주차 | Ralphthon 실행 + 중간 보고 | 2026-06-10 |
| 15주차 | 통합 테스트 + 발표 자료 제출 | 2026-06-17 |
| 16주차 | 최종 발표 + 동료 평가 | 2026-06-24 |

## 평가 기준

캡스톤은 총 30점 (전체 성적의 30%). 별도로 **8주차 계획서는 중간 프로젝트 20%**로 집계된다.

| 항목 | 배점 | 설명 |
|------|------|------|
| 기술적 완성도 | 12점 | 에이전트 파이프라인 실제 동작 |
| 문제 적합성 | 6점 | 주제 선정의 적절성과 해결 효과 |
| 발표 품질 | 6점 | 명확한 설명, 라이브 데모 |
| 동료 평가 | 6점 | 다른 학생 발표/결과물에 대한 상호 평가 |

## 제출 구조

```
capstone/
└── projects/
    └── [학번]/
        ├── proposal.md        # 프로젝트 계획서 (8주차)
        ├── design.md          # 아키텍처 설계 문서 (13주차)
        ├── progress-week14.md # 중간 진행 보고 (14주차)
        ├── README.md          # 프로젝트 최종 문서 (15주차)
        ├── report.md          # 최종 보고서 (15주차)
        ├── links.md           # 발표 자료 + 데모 영상 외부 링크 (15주차)
        └── src/                # 소스 코드
```

> **발표 자료·데모 영상은 외부 링크로 제출**: 저장소 용량 관리를 위해 `.pdf`·`.mp4` 파일은 직접 커밋하지 않고 `links.md`에 **Google Drive / YouTube / Figma** 등 외부 URL을 기록한다. `.gitignore`에서 `*.pdf`, `*.mp4`가 차단되어 있다.
