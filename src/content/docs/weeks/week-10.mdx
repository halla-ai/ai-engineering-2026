## 이론 (Theory)

### DeepSeek-Coder-V2 개요

DeepSeek-Coder-V2는 236B 파라미터의 MoE(Mixture-of-Experts) 모델로, 코딩 작업에서 GPT-4o 수준의 성능을 보여준다.

| 특성 | 수치 |
|------|------|
| 총 파라미터 | 236B |
| 활성 파라미터 | 21B (추론 시) |
| 컨텍스트 창 | 128K 토큰 |
| 지원 언어 | 338개 프로그래밍 언어 |
| 라이선스 | DeepSeek License (상용 허용) |

<Aside type="tip" title="MoE 아키텍처의 효율성">
236B 파라미터 모두를 사용하는 대신, 각 토큰 처리 시 21B만 활성화. H100 1개로도 운영 가능.
</Aside>

### 오픈소스 vs 상용 API 비교

<CardGrid>
  <Card title="상용 API (Claude, GPT-4)" icon="external">
    - 장점: 즉시 사용, 유지보수 불필요
    - 단점: 데이터 프라이버시, 비용 예측 불가, API 제한
    - 비용: ~$15/1M 토큰 (input)
  </Card>
  <Card title="오픈소스 (DeepSeek + vLLM)" icon="laptop">
    - 장점: 완전한 제어, 데이터 내부 보관, 비용 예측 가능
    - 단점: 초기 설정 비용, 유지보수 필요
    - 비용: H100 서버 비용만 (토큰당 ~$0.001)
  </Card>
</CardGrid>

## 실습 (Practicum)

### vLLM으로 DeepSeek 배포

<Steps>
1. **DGX 서버에서 vLLM 설치**

   ```bash
   # MIG 슬라이스 내에서
   pip install vllm
   ```

2. **모델 다운로드 및 서버 실행**

   ```bash
   python -m vllm.entrypoints.openai.api_server \
     --model deepseek-ai/DeepSeek-Coder-V2-Lite-Instruct \
     --tensor-parallel-size 1 \
     --max-model-len 32768 \
     --port 8000
   ```

3. **OpenAI 호환 API로 테스트**

   ```python
   import openai

   client = openai.OpenAI(
       base_url="http://localhost:8000/v1",
       api_key="token-abc"
   )

   response = client.chat.completions.create(
       model="deepseek-ai/DeepSeek-Coder-V2-Lite-Instruct",
       messages=[{"role": "user", "content": "Python으로 퀵소트 구현해줘"}]
   )
   print(response.choices[0].message.content)
   ```

4. **성능 벤치마크**

   ```bash
   # 처리량 벤치마크
   python -m vllm.benchmarks.benchmark_throughput \
     --model deepseek-ai/DeepSeek-Coder-V2-Lite-Instruct \
     --num-prompts 100 \
     --input-len 512 \
     --output-len 128
   ```
</Steps>
