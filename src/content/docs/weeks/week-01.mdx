## 이론 (Theory)

### AI 시스템의 패러다임 전환

2025–2026년 AI 산업은 근본적인 전환점을 맞이했다. LLM을 단순한 텍스트 생성 도구로 활용하던 시대가 끝나고, **자율 에이전틱 시스템**이 소프트웨어 개발의 주류가 되었다.

<CardGrid>
  <Card title="Before (2023–2024)" icon="close">
    - LLM = 코드 제안 도구 (Copilot)
    - 인간이 모든 코드를 검토·승인
    - AI는 수동적 "오라클"
  </Card>
  <Card title="After (2025–2026)" icon="approve-check-circle">
    - LLM = 자율 실행 에이전트
    - 인간은 전략적 감독자
    - AI가 파일 수정·테스트·배포 직접 수행
  </Card>
</CardGrid>

### HITL vs HOTL 거버넌스 아키텍처

| 구분 | Human-in-the-Loop (HITL) | Human-on-the-Loop (HOTL) |
|------|--------------------------|--------------------------|
| **인간 역할** | 순차적 게이트키퍼 | 전략적 감독자 |
| **AI 실행** | 인간 승인 후 실행 | 자율 실행, 인간은 모니터링 |
| **속도** | 낮음 (병목 발생) | 높음 (병렬 처리 가능) |
| **활용 예** | 프로덕션 DB 수정 | CI/CD 파이프라인, 코드 생성 |
| **규제 요건** | EU AI Act 고위험 시스템 | 텔레메트리 + 감사 로그 필수 |

<Aside type="note" title="Human-in-Command (HIC)">
HOTL보다 상위 개념으로, AI가 실행하는 동안 인간이 **전체 전략과 경계 조건**을 설정하는 아키텍처. 이 강의의 궁극적 목표는 HIC 엔지니어를 양성하는 것이다.
</Aside>

### 로컬 환경 준비

<Steps>
1. **Node.js 20 LTS 설치**

   ```bash
   # macOS (Homebrew)
   brew install node@20

   # Ubuntu/Debian
   curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
   sudo apt-get install -y nodejs
   ```

2. **AI 코딩 CLI 도구 설치**

   <Tabs>
     <TabItem label="Claude Code (권장)">
       다음 명령어로 Claude Code를 설치합니다.
       <Tabs>
         <TabItem label="Native(recommend)">
           ```bash
           # macOS, Linux, WSL
           curl -fsSL https://claude.ai/install.sh | bash
           ```
         </TabItem>
         <TabItem label="Homebrew">
           ```bash
           brew install claude-code
           ```
         </TabItem>
         <TabItem label="npm">
           ```bash
           npm install -g @anthropic-ai/claude-code
           ```
         </TabItem>
       </Tabs>

       그런 다음 프로젝트에서 Claude Code를 시작합니다:
       ```bash
       claude --version

       cd your-project
       claude
       ```
     </TabItem>
     <TabItem label="Gemini CLI">
       ```bash
       npm install -g @google/gemini-cli
       gemini --version
       ```
     </TabItem>
     <TabItem label="Codex CLI">
       ```bash
       npm install -g @openai/codex
       codex --version
       ```
     </TabItem>
     <TabItem label="OpenCode">
       ```bash
       brew install opencode
       ```
     </TabItem>
   </Tabs>

   <Aside type="note">도구별 상세 비교는 [AI 코딩 도구 선택 가이드](/reference/tool-selection) 참조.</Aside>

3. **API 키 설정**

   <Tabs>
     <TabItem label="Claude Code">
       ```bash
       export ANTHROPIC_API_KEY="sk-ant-..."
       # ~/.bashrc 또는 ~/.zshrc에 추가
       ```
     </TabItem>
     <TabItem label="Gemini CLI">
       ```bash
       export GEMINI_API_KEY="..."
       # 또는 첫 실행 시 브라우저 OAuth 인증
       ```
     </TabItem>
     <TabItem label="Codex CLI">
       ```bash
       export OPENAI_API_KEY="sk-..."
       ```
     </TabItem>
     <TabItem label="OpenCode">
       ```bash
       # 사용할 모델 제공자에 따라 설정
       export OPENAI_API_KEY="sk-..."  # OpenAI 모델 사용 시
       ```
     </TabItem>
   </Tabs>

4. **첫 번째 실행 테스트**

   <Tabs>
     <TabItem label="Claude Code">
       ```bash
       mkdir ~/test-project && cd ~/test-project
       claude "Hello! 이 디렉토리에 간단한 Python hello world 파일을 만들어줘."
       ```
     </TabItem>
     <TabItem label="Gemini CLI">
       ```bash
       mkdir ~/test-project && cd ~/test-project
       gemini
       # 대화형 모드에서: "이 디렉토리에 간단한 Python hello world 파일을 만들어줘."
       ```
     </TabItem>
     <TabItem label="Codex CLI">
       ```bash
       mkdir ~/test-project && cd ~/test-project
       codex "이 디렉토리에 간단한 Python hello world 파일을 만들어줘."
       ```
     </TabItem>
     <TabItem label="OpenCode">
       ```bash
       mkdir ~/test-project && cd ~/test-project
       opencode
       # TUI에서: "이 디렉토리에 간단한 Python hello world 파일을 만들어줘."
       ```
     </TabItem>
   </Tabs>
</Steps>
